学 Git 的核心心得，三点足矣：

**1. 理解核心逻辑**
Git 不是“保存文件”，而是**记录快照与分支指针的移动**。每次提交都是项目的一个完整快照，分支只是指向某个提交的指针。想通这一点，就能理解版本如何“穿梭”。

**2. 掌握最小命令集**
- **记录状态**：`add`（暂存）→ `commit`（提交）
- **修复错误**：`checkout`（恢复文件）、`reset`（重置提交）、`revert`（撤销提交）
- **同步协作**：`pull`（拉取）= `fetch`（获取） + `merge`（合并），`push`（推送）
- **分支管理**：`branch`（查看/创建）、`checkout`/`switch`（切换）、`merge`（合并）

**3. 接受“先混乱后清晰”的过程**
初期遇到冲突、丢失代码是必然的。关键两步：
- **遇错不急**：用 `git status` 和 `git log` 看清状态
- **大胆尝试**：在安全副本中练习 `rebase`、`cherry-pick` 等操作

**最终体会**：Git 本质是**时间管理工具**。学会为每个功能开分支、写清晰的提交信息、保持主线整洁，比精通所有命令更重要。真正的熟练，始于接受自己会搞乱，并相信总能找回代码。
